---
name: write-tests
description: Use this skill when the task requires adding or updating automated tests for new or existing behavior.
---

# Goal
Improve confidence in behavior through focused automated tests.

# Steps
1. Identify the behavior that must be protected.
2. Prefer existing test patterns and helpers.
3. Add the smallest useful set of tests.
4. Keep tests readable and deterministic.
5. Note any gaps that could not be covered.

# Constraints
- Do not add slow or brittle tests without justification.
- Keep test scope close to the change.

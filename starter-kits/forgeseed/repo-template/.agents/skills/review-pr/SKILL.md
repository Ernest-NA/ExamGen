# Goal
Produce a concise and useful technical review.

# Steps
1. Inspect the changed files.
2. Identify correctness or design risks.
3. Check whether validation is adequate.
4. Note missing docs or tests.
5. Summarize findings by priority.

# Constraints
- Prefer actionable comments.
- Separate blocking issues from suggestions.

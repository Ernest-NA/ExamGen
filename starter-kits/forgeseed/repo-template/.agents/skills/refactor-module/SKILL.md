---
name: refactor-module
description: Use this skill when reorganizing an existing module to improve clarity, cohesion, or maintainability.
---

# Goal
Improve internal structure with careful, reviewable changes.

# Steps
1. Identify the maintainability problem.
2. Refactor in small increments.
3. Keep interfaces stable unless the task allows a change.
4. Update tests or add coverage if needed.
5. Summarize trade-offs and follow-up items.

# Constraints
- Avoid mixing refactor work with unrelated feature work.
- Prefer incremental changes over broad rewrites.

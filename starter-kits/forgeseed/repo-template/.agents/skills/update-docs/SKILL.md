---
name: update-docs
description: Use this skill when project documentation must be added or updated to reflect a technical or product change.
---

# Goal
Keep repository documentation aligned with the current system and workflow.

# Steps
1. Identify which docs are affected.
2. Update the smallest set of documents that keeps the repo accurate.
3. Prefer concrete instructions over vague summaries.
4. Call out assumptions or known gaps.
5. Link related code or process changes where useful.

# Constraints
- Avoid duplicating documentation across multiple files.
- Preserve existing terminology unless there is a clear reason to change it.
